01 / PRINCIPLES

动效原则

动效不是装饰，而是信息。每一段动画都应该回答同一个问题： 「我在这里想让用户感知到什么？」

PRINCIPLE · 01

Purposeful · 有目的

每个动画都应有明确的意图：引导注意力、提供反馈、或建立空间关系。无目的的动效只会分散用户。

PRINCIPLE · 02

Natural · 符合物理直觉

动画应模拟现实世界的惯性、弹性与重力。缓入缓出代替匀速，让运动更有重量感。

PRINCIPLE · 03

Fast · 快速响应

用户交互后 100ms 内必须有反馈。所有微交互都应在视觉上即时响应，不拖慢用户。

PRINCIPLE · 04

Subtle · 精致克制

动效应服务内容，不喧宾夺主。过慢、过夸张、过频繁的动画都会将用户的注意力从任务中拉开。

02 / EASING

缓动曲线

所有动画都使用标准的 cubic-bezier 曲线。悬停或点击卡片查看小球的实际运动。

Linear · 线性

linear

用于持续旋转的加载指示器、循环进度条等匀速运动。

Ease Out · 快出

cubic-bezier(0.16, 1, 0.3, 1)

默认曲线。 用于大多数出现动画、入场、展开，给人「快速响应」的印象。

点击卡片播放

Ease In · 慢出

cubic-bezier(0.4, 0, 1, 1)

用于离场、收起、元素消失的动画。让消失显得「有重量」而不突兀。

点击卡片播放

Ease In Out · 对称

cubic-bezier(0.4, 0, 0.2, 1)

用于来回循环的动画：标签切换、轮播、开关在两个状态之间的切换。

Spring · 弹性

cubic-bezier(0.34, 1.56, 0.64, 1)

用于强调「成功」「完成」等正向反馈事件，或需要吸引注意力的弹出。

Button · Hover & Active

Card · Hover lift

Input · Focus ring

Switch · Toggle

07 / SCROLLING

滚动动效

滚动驱动的效果应提升阅读体验，而非影响性能。以下是三类常见的滚动相关动效。

A · Sticky Header · 滚动收缩

页面顶部 header 会在向下滚动时略微收缩（见顶部导航）。下方是一个小型容器内的 sticky 头演示：

📌 始终可见的标题

当内部内容滚动时，此标题始终贴在容器顶部，帮助用户保持上下文。

在实际应用中，sticky 头常配合高度收缩 + 阴影出现效果，以节约纵向空间。

ZenDraw 使用 position: sticky 而非 JS 计算位置，性能更好。

关键实现：当 window.scrollY > 0 时给 header 添加 .is-scrolled 类，触发高度与阴影的 CSS 过渡。

这样 header 仅通过 CSS 变量计算即可获得「被滚动过」的视觉状态。

滚动此区域，你会发现标题始终贴在容器顶部。这是一种低成本的内容锚点。

继续填充内容以演示滚动行为：动效需要和内容量匹配，不应在短页面中滚动触发。

对于长页面，滚动动效会成为阅读节奏的一部分。

B · Reveal On Scroll · 视口显现

向下滚动时，以下卡片会以 80ms 为间隔依次滑入。使用 IntersectionObserver 触发。

Reveal Card · 01

在用户滚动到视口时，从下方 24px 处渐显向上移动到正常位置。

Reveal Card · 02

列表项建议使用 staggered delay（每项 +80ms）来建立节奏。

Reveal Card · 03

但 delay 不应超过 200ms，否则用户会感觉动画变慢。

Reveal Card · 04

一旦元素已出现，再次滚动经过时不应重新播放动画，避免闪烁。

C · Reading Progress · 阅读进度条

页面顶部 header 的下方有一条细进度线，滚动页面时实时指示当前阅读位置（0–100%）。

实现方式：监听 scroll 事件 → 计算 window.scrollY / (docHeight - viewportHeight) → 以百分比设置进度条的宽度。使用 rAF 节流避免性能问题。

JS · reveal on scroll
const io = new IntersectionObserver((entries) => {
  entries.forEach((entry, i) => {
    if (entry.isIntersecting) {
      entry.target.classList.add('is-visible');
      io.unobserve(entry.target);
    }
  });
}, { threshold: 0.15 });

document.querySelectorAll('.reveal-item').forEach(el => io.observe(el));
08 / A11Y

可访问性 · Reduced Motion

动效可能触发前庭功能障碍用户（vestibular disorder）的不适。 我们必须在系统层面尊重 prefers-reduced-motion: reduce。

正常模式 · Motion On

所有过渡、动画、变换正常播放。悬停按钮观察。

减动效模式 · Reduced

动画时长设为 0，移除 transform，仅保留必要的 opacity 过渡。

强制减动效
切换整个页面的动效。若系统设置了 prefers-reduced-motion: reduce，ZenDraw 默认会自动启用此模式。 
CSS · reduced motion
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.001ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.001ms !important;
    scroll-behavior: auto !important;
  }
}

/* 保留必要的 opacity 过渡，以确保状态切换仍可见，
   避免依赖动效传达关键信息。 */

实践清单

  • 永远不要通过动效来传达关键信息。例如，错误消息必须以文字、颜色等其他方式可见。
  • 提供「开关」。允许用户在应用内关闭动效，并持久化该偏好。
  • 尊重系统设置。检测 window.matchMedia('(prefers-reduced-motion: reduce)') 并默认启用减动效模式。
  • 避免闪烁。超过 3 次/秒的闪烁可能触发癫痫。避免 CSS 闪烁动画。
  • 视口内的内容不要自动移动。轮播、走马灯需要用户手动触发，或提供暂停控制。
  • 使用 opacity 而非 display。当需要隐藏内容时，opacity 的过渡比 display 切换更平滑且易降级。
