组件库规范

一份面向 ZenDraw 开发者与设计师的组件库参考文档。 基于 shadcn/ui 的设计语言，包含基础组件、复合组件、业务组件以及常见设计模式。 所有组件均支持浅色 / 深色主题切换。

1. 基础组件 · Base Components

不可再分的原子级 UI 元素。所有复合组件均由此构建，保持统一的视觉语言与交互范式。

Button

#button

触发操作的主要交互元素。支持 6 种视觉变体与 4 种尺寸。

variants · default sizes
sizes · states
Size
State
✓ Do

主操作按钮使用 primary，置于视觉焦点位置。

次级操作使用 ghost 或 outline，形成层级。

按钮文案使用动词开头，如 "开始抽奖"、"保存设置"。

✕ Don't

同一行放置超过 2 个 primary 按钮。

使用 destructive 作为常规操作按钮。

按钮文案过长超过 10 个汉字。

尺寸
最小点击区域 32×32px（符合 WCAG 2.5.5 Target Size）

  disabled={false}

  {children}
</Button>

Input

#input

单行文本输入框。支持图标前缀、禁用与无效状态。

variants
✓ Do

始终为输入框提供清晰的 Label 或占位符。

数值输入时配合单位提示，如 "范围 (1 - 100)"。

错误状态配合下方错误文案一起展示。

✕ Don't

仅依赖 placeholder 作为输入说明（会在输入时消失）。

混用图标、前缀、后缀使输入框显得拥挤。

键盘
Tab 聚焦 · Esc 清除 · 方向键移动光标
ARIA
aria-label / aria-describedby 关联帮助文本
错误
使用 aria-invalid="true" 与 aria-errormessage
标签
必须使用显式 <label htmlFor>，屏幕阅读器才可正确朗读
<Input
  type="number"
  placeholder="请输入最大值"
  value={value}
  onChange={onChange}
  disabled={false}
/>

// 带前缀图标
<div className="input">
  <SearchIcon className="inp-icon" />
  <input placeholder="搜索..." />
</div>

Textarea

#textarea

多行文本输入。用于备注、配置描述、批量数字粘贴等场景。

variants
✓ Do

允许纵向 resize，方便用户自行调整大小。

配合 maxLength 或字符计数器限制输入长度。

✕ Don't

对于简短输入（< 1 行）使用 Textarea，应改用 Input。

在移动端设置过高的默认高度。

键盘
Enter 换行 · Tab 切换焦点（可配置为缩进）
ARIA
需与 <label> 关联 · aria-multiline="true"
<Textarea
  placeholder="粘贴抽奖名单，每行一个..."
  rows={5}
  maxLength={2000}
  onChange={onChange}
/>

Badge

#badge

状态、分类、数值的小型标记组件。

variants
Default Secondary Outline Error 成功 警告 提示
✓ Do

使用 Badge 标记状态（进行中、已完成）或分类。

✕ Don't

在单个 Badge 中放置过长文本（建议 < 6 字）。

#tabs

标签页切换组件。用于在同一区域展示多组并列内容。

#form-field

normal · invalid
最小值 *
范围 1 - 9999，不能大于最大值

4. 设计模式

常见交互场景的参考实现。帮助开发者在不同模块保持一致的用户体验。

表单验证

#form-validation

实时验证 (实时反馈) 与 提交验证 (点击提交后统一验证) 两种模式。

instant vs. on-submit
实时验证 · Instant
最小值
✓ 数值合法
最大值
必须 ≥ 1
提交验证 · On Submit
最小值
最大值
请修正以下错误
最大值必须大于或等于 1
✓ Do

复杂表单使用实时验证，避免用户在提交时发现大量错误。

简单设置类表单使用提交验证，保持界面清爽。

✕ Don't

两种模式混用，让用户感到困惑。

错误消息仅用颜色标识（缺少图标或文本标签）。

空状态

#empty-state

当没有历史记录或数据时的显示方式。

empty state
∅

暂无抽奖记录

点击下方按钮开始第一次抽奖吧

✓ Do

提供简明的图标 + 标题 + 描述 三段式结构。

在合适情况下提供"首次操作"的引导按钮。

✕ Don't

仅用空白区域代替空状态，让用户感到困惑。

使用过多文案或夸张的插画。
