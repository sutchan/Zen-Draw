Interaction

交互规范

统一的交互模式，确保一致的心理预期。

悬停反馈

  • 按钮悬停：translateY(-1px) + 阴影加深
  • 图标按钮悬停：背景色切换至 --bg-muted
  • 卡片悬停：Y 位移 + box-shadow: lg + border 强化
  • 动画时长：150-250ms，缓动：cubic-bezier(0.16, 1, 0.3, 1)
1
2
3

空状态 / 错误状态

  • 空状态：装饰性图标 + 友好文案 + CTA
  • 错误状态：danger 色 + 明确错误说明
  • 表单错误：input 边框变红 + 下方错误描述
  • 加载状态：Skeleton 占位图，不使用 spinner

键盘操作

  • Enter：确认 / 触发默认操作
  • Esc：关闭对话框 / 清空当前输入
  • Tab：按视觉顺序循环聚焦（焦点始终可见）
  • 所有可点击元素必须可键盘聚焦并可见焦点指示

动画原则

  • 动画必须服务于信息传达，不为动而动
  • 尊重 prefers-reduced-motion
  • 出/入动画：淡入淡出 + 轻微位移，0.3-0.4s
  • 结果数字：快速弹出 + 弹簧物理，stiffness 260

Accessibility

无障碍设计

让所有用户都能顺畅使用产品。

⌨

键盘导航

所有交互元素均可通过 Tab 键聚焦，并提供可见的焦点指示环。Enter 触发，Esc 取消。

🎯

颜色对比度

所有文本与背景的对比度达到 WCAG 2.1 AA 标准 (4.5:1)，大字达到 3:1。

🔊

屏幕阅读器

语义化 HTML、ARIA 属性、aria-live 区域，确保屏幕阅读器能正确传达状态变化。

🧭

动效偏好

检测 prefers-reduced-motion，尊重用户的动效偏好设置，对敏感用户关闭非必要动画。

🌓

深色模式

完整的浅色 / 深色 / 跟随系统三态切换，所有组件均在两种模式下表现良好。

📐

触摸目标

所有可点击元素最小尺寸为 40×40px，满足 WCAG 对触摸目标大小的 24×24 CSS 像素建议。
